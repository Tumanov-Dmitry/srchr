# Дизайн-система SRCHR 2.0

## Простыми словами

Дизайн-система — это общий набор кнопок, полей, карточек, окон, иконок, графиков и правил анимации. Благодаря ей разные разделы SRCHR выглядят и ведут себя как один продукт.

## Зачем нужен модуль

Единые компоненты ускоряют разработку, уменьшают визуальные ошибки и позволяют менять стиль платформы без ручной перекраски каждой страницы.

## Что можно делать

- собирать интерфейсы из shadcn/ui-компонентов;
- использовать продуктовые блоки из `components/srchr`;
- проверять состояния компонентов на `/dev/ui`;
- использовать Hugeicons вместо разрозненных SVG;
- строить графики аналитики в общем стиле;
- применять локально размещённый Geist без внешней загрузки шрифта.

## Визуальный язык

Тема SRCHR использует стиль NOVA: светлый нейтральный фон, зелёный основной цвет, спокойные вторичные состояния, крупные, но контролируемые скругления и лёгкие границы вместо тяжёлых теней.

Анимации диалогов, popover и уведомлений должны быть короткими и плавными. Мобильные всплывающие элементы появляются снизу; компактные выборы не разворачиваются на весь экран без необходимости.

## Компоненты

- `components/ui` — базовые компоненты shadcn/ui;
- `components/srchr` — композиции и карточки сущностей SRCHR;
- `app/dev/ui/page.tsx` — интерактивный playground;
- `app/globals.css` — токены темы и глобальные правила;
- `lib/utils.ts` — объединение классов и общие UI-утилиты.

Нативные поля даты и числа стилизуются так, чтобы системные индикаторы не конфликтовали с иконками и размерами дизайн-системы.

## Правило развития

Новая страница сначала использует существующий компонент. Новый базовый компонент добавляется только когда текущего набора недостаточно, после чего его состояния показываются в `/dev/ui`.

## Ограничения

Миграция старых экранов выполняется постепенно. Визуальное обновление не должно менять бизнес-логику, API или права доступа.
