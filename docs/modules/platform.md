# Платформа и роли

## Назначение

SRCHR связывает заказчиков, агентства-подрядчиков и экспертов. Пользователь является владельцем личного кабинета, но публичные профили и коммерческие действия выполняются через отдельные сущности.

## Модель

- `profiles` хранит пользовательский профиль и уровень доступа.
- `organizations` хранит компании и агентства.
- `organization_members` связывает пользователей с несколькими организациями.
- `expert_profiles` добавляет пользователю необязательный публичный профиль эксперта.

Организация не заменяет эксперта, а эксперт не заменяет организацию.

## Роли

- `guest` — зарегистрированный пользователь без завершённой настройки.
- `contractor` — исторический тип аккаунта; рабочая сущность подрядчика определяется организацией.
- `client` — исторический тип аккаунта; рабочая сущность заказчика определяется организацией.
- `admin`, `moderator`, `super_admin` — административный доступ.

Тип `both` не используется в новой бизнес-логике.

## Принципы доступа

Права проверяются на сервере. Доступ к организации определяется членством и ролью `owner`, `admin`, `editor` или `member`. Публичные страницы показывают только опубликованные сущности.

## Ограничения

Часть старых записей может содержать исторические значения типов аккаунта. Их нельзя массово изменять без отдельного плана миграции.
