# Авторизация и онбординг

## Простыми словами

Это входная дверь в SRCHR. Модуль регистрирует пользователя, запоминает его сессию и помогает впервые настроить аккаунт.

## Зачем нужен модуль

Без авторизации невозможно надёжно определить, кому принадлежат профиль, организация, материал, задача или избранное. Онбординг помогает не бросать нового пользователя в пустой кабинет.

## Что можно делать

- зарегистрироваться и войти;
- выйти из аккаунта;
- восстановить текущую сессию после обновления страницы;
- выбрать первый рабочий сценарий;
- создать экспертный профиль или организацию;
- перейти в подходящий личный кабинет.

## Назначение

Модуль отвечает за регистрацию, вход, выход, получение текущего пользователя и первичную настройку профиля.

## Основной сценарий

1. Пользователь регистрируется через Supabase Auth.
2. Для него создаётся запись в `profiles`.
3. Незавершённый пользователь направляется в onboarding.
4. Пользователь создаёт экспертный профиль или организацию нужного типа.
5. После завершения он попадает в личный кабинет.

Онбординг можно пропустить. В этом случае профиль корректно помечается завершённым без обязательного создания эксперта или организации, а пользователь попадает в базовый dashboard.

## Бизнес-правила

- Сессия хранится в cookie и читается серверным Supabase-клиентом.
- Закрытые страницы не должны доверять только клиентскому состоянию.
- Административная роль не выбирается пользователем самостоятельно.
- Создание профиля должно быть идемпотентным: повторный вызов не создаёт дубль.
- Пропуск онбординга не назначает пользователю фиктивную роль и не создаёт организацию.
- Возврат после входа допускается только на внутренний безопасный маршрут.
