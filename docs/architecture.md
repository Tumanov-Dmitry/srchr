# Архитектура SRCHR

База знаний хранится рядом с кодом и обновляется вместе с функциональностью.

## Правило обновления

При изменении бизнес-логики обновляется только документ затронутого модуля и его дата в `docs/modules/index.json`. Документы не должны копировать большие фрагменты кода.

## Структура

- `docs/modules/index.json` — компактный индекс, статусы и зависимости.
- `docs/modules/*.md` — актуальная логика отдельных модулей.
- `/admin/knowledge` — интерфейс чтения для администраторов.

## Статусы

- `active` — используется в продукте;
- `beta` — работает, но контракт ещё может измениться;
- `planned` — описан, но не реализован;
- `deprecated` — оставлен для совместимости.

## Шаблон модуля

Каждый документ начинается с продуктового объяснения:

- что делает модуль простыми словами;
- зачем он нужен пользователю и бизнесу;
- какие возможности предоставляет.

После этого документ фиксирует бизнес-правила, статусы, доступ, таблицы, маршруты, реализацию, связи и ограничения.
