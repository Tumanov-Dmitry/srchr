# Analytics

## Purpose

Analytics provides visibility into platform usage, object performance, and future recommendation signals.

## Current Model

Analytics is event-based.

Typical events:

- profile views;
- material views;
- event views;
- tender views;
- author clicks;
- organization clicks;
- QR scans;
- favorite actions;
- response actions.

## Product Uses

Analytics supports:

- user dashboard insights;
- admin health monitoring;
- public insights where allowed;
- content performance;
- recommendation inputs;
- reputation and quality signals.

## Privacy Rules

- Public analytics should be aggregated and safe.
- User-specific analytics require authentication and ownership checks.
- Admin analytics can be broader but must remain server-authorized.
- Do not expose raw sensitive user behavior to guests.

## Future Direction

Analytics can support:

- conversion funnels;
- profile performance;
- content performance;
- event promotion reports;
- tender response analytics;
- recommendation training data;
- abuse detection.
