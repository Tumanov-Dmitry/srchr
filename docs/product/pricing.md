# Pricing

## Status

Pricing is not final. Treat current subscription/payment code as foundation or placeholder unless a task explicitly changes monetization behavior.

## Current Product Rule

- Expert profile is free at first stage.
- Basic profile creation should remain available.
- Payments and subscriptions should not block MVP flows unless explicitly implemented.
- Promotion can be manually controlled by admin before automated payment is connected.

## Future Pricing Axes

Possible pricing dimensions:

- contractor profile tier;
- number of published cases/articles;
- lead/contact access;
- response limits;
- promoted placement;
- advanced analytics;
- verification;
- team seats;
- event promotion;
- AI recommendations or shortlist tools.

## Guardrails

- Do not create hidden hard paywalls without product decision.
- Do not change subscription semantics from unrelated module tasks.
- Do not couple expert profile to paid subscription in MVP.
- Keep pricing logic isolated from catalog rendering where possible.
