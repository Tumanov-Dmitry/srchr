# Product Documentation

This section describes product intent and business rules that should guide implementation decisions.

## Files

- `vision.md` - product direction and non-goals.
- `roles.md` - users, experts, organizations, contractors, clients, admins.
- `business-model.md` - value model and monetization direction.
- `pricing.md` - current pricing assumptions and future axes.
- `reputation.md` - trust and reputation principles.
- `recommendation-system.md` - recommendation inputs and guardrails.
- `search.md` - search principles and future direction.
- `analytics.md` - analytics principles and privacy rules.

## Rule

Product docs describe intended behavior. If implementation differs, either update the docs during the task or explicitly report the mismatch.
